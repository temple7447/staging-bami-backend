# Payment Verification - Frontend Integration Guide\n\n**Status**: ✅ Production Ready  \n**Last Updated**: November 2024\n\n---\n\n## Overview\n\nThis guide explains how to integrate payment verification in your frontend application. After a tenant completes payment on Paystack, your frontend should verify the payment status with the backend.\n\n---\n\n## Payment Flow\n\n```\n1. Admin initiates payment\n   ↓\n2. Backend generates Paystack checkout link\n   ↓\n3. Frontend redirects tenant to Paystack checkout\n   ↓\n4. Tenant completes payment on Paystack\n   ↓\n5. Paystack redirects to frontend with reference\n   ↓\n6. Frontend calls verification endpoint\n   ↓\n7. Backend verifies with Paystack API\n   ↓\n8. Payment marked as \"completed\" in database\n   ↓\n9. Frontend displays success/failure message\n```\n\n---\n\n## API Endpoints\n\n### Initiate Payment\n```bash\nPOST /api/payments/{payment-type}\nAuthorization: Bearer {TOKEN}\nContent-Type: application/json\n\n{\n  \"tenantId\": \"690e3ff75b552b1489bd3672\",\n  \"amount\": 50000,\n  \"description\": \"Monthly rent\"\n}\n\nResponse (201):\n{\n  \"success\": true,\n  \"data\": {\n    \"paymentId\": \"507f1f77bcf86cd799439012\",\n    \"paymentLink\": \"https://checkout.paystack.com/c1e3q5jvmjost5w\",\n    \"reference\": \"PAY-1762804019979-eao85mwxb\",\n    \"accessCode\": \"c1e3q5jvmjost5w\",\n    \"amount\": 50000\n  }\n}\n```\n\n### Verify Payment ✅ NEW\n```bash\nGET /api/payments/verify/{reference}\n\nResponse (200):\n{\n  \"success\": true,\n  \"message\": \"Payment verification successful - Status: success\",\n  \"data\": {\n    \"paymentId\": \"507f1f77bcf86cd799439012\",\n    \"reference\": \"PAY-1762804019979-eao85mwxb\",\n    \"status\": \"completed\",\n    \"paystackStatus\": \"success\",\n    \"amount\": 50000,\n    \"currency\": \"NGN\",\n    \"type\": \"caution_fee\",\n    \"paidAt\": \"2024-11-10T19:45:23.000Z\",\n    \"tenant\": { ... },\n    \"estate\": { ... }\n  }\n}\n```\n\n---\n\n## Frontend Implementation\n\n### Step 1: Initiate Payment (Vue.js Example)\n\n```javascript\n// Vue component method\nasync initiatePayment() {\n  try {\n    const response = await fetch('http://localhost:5000/api/payments/rent', {\n      method: 'POST',\n      headers: {\n        'Authorization': `Bearer ${this.authToken}`,\n        'Content-Type': 'application/json'\n      },\n      body: JSON.stringify({\n        tenantId: this.selectedTenant._id,\n        amount: 150000,\n        description: 'November 2024 rent payment'\n      })\n    });\n\n    const result = await response.json();\n\n    if (result.success) {\n      // Store payment details for verification\n      this.currentPayment = {\n        paymentId: result.data.paymentId,\n        reference: result.data.reference,\n        amount: result.data.amount,\n        type: 'rent'\n      };\n\n      // Redirect to Paystack checkout\n      window.location.href = result.data.paymentLink;\n    } else {\n      this.showError(result.message);\n    }\n  } catch (error) {\n    console.error('Payment initiation error:', error);\n    this.showError('Failed to initiate payment');\n  }\n}\n```\n\n### Step 2: Verify Payment After Redirect\n\n```javascript\n// Called when redirected back from Paystack\n// Get reference from URL: http://localhost:8080/payment-success?reference=PAY-...\n\nasync mounted() {\n  const urlParams = new URLSearchParams(window.location.search);\n  const reference = urlParams.get('reference');\n\n  if (reference) {\n    await this.verifyPayment(reference);\n  }\n}\n\nasync verifyPayment(reference) {\n  try {\n    this.verifying = true;\n\n    const response = await fetch(\n      `http://localhost:5000/api/payments/verify/${reference}`\n    );\n\n    const result = await response.json();\n\n    if (result.success) {\n      // Payment verified successfully\n      if (result.data.status === 'completed') {\n        this.paymentSuccess = true;\n        this.paymentResult = result.data;\n        this.showSuccess(\n          `Payment of ₦${result.data.amount.toLocaleString()} confirmed!`\n        );\n      } else {\n        this.paymentFailed = true;\n        this.failureReason = `Payment status: ${result.data.paystackStatus}`;\n        this.showError('Payment verification failed');\n      }\n    } else {\n      this.paymentFailed = true;\n      this.failureReason = result.error;\n      this.showError(result.message);\n    }\n  } catch (error) {\n    console.error('Payment verification error:', error);\n    this.paymentFailed = true;\n    this.failureReason = 'Network error during verification';\n    this.showError('Failed to verify payment');\n  } finally {\n    this.verifying = false;\n  }\n}\n```\n\n### Step 3: Display Payment Result\n\n```html\n<template>\n  <div v-if=\"verifying\" class=\"payment-verifying\">\n    <div class=\"spinner\"></div>\n    <p>Verifying your payment...</p>\n  </div>\n\n  <div v-else-if=\"paymentSuccess\" class=\"payment-success\">\n    <div class=\"success-icon\">✅</div>\n    <h2>Payment Successful!</h2>\n    <div class=\"payment-details\">\n      <p><strong>Amount:</strong> ₦{{ paymentResult.amount.toLocaleString() }}</p>\n      <p><strong>Reference:</strong> {{ paymentResult.reference }}</p>\n      <p><strong>Tenant:</strong> {{ paymentResult.tenant.name }}</p>\n      <p><strong>Unit:</strong> {{ paymentResult.tenant.unit }}</p>\n      <p><strong>Date:</strong> {{ new Date(paymentResult.paidAt).toLocaleString('en-NG') }}</p>\n    </div>\n    <button @click=\"goToDashboard\" class=\"btn btn-primary\">\n      Go to Dashboard\n    </button>\n  </div>\n\n  <div v-else-if=\"paymentFailed\" class=\"payment-failed\">\n    <div class=\"error-icon\">❌</div>\n    <h2>Payment Failed</h2>\n    <p class=\"error-message\">{{ failureReason }}</p>\n    <div class=\"action-buttons\">\n      <button @click=\"retryPayment\" class=\"btn btn-primary\">\n        Retry Payment\n      </button>\n      <button @click=\"goToDashboard\" class=\"btn btn-secondary\">\n        Go to Dashboard\n      </button>\n    </div>\n  </div>\n</template>\n```\n\n---\n\n## React Implementation\n\n```javascript\nimport { useEffect, useState } from 'react';\nimport { useSearchParams, useNavigate } from 'react-router-dom';\n\nexport function PaymentVerification() {\n  const [searchParams] = useSearchParams();\n  const navigate = useNavigate();\n  const [loading, setLoading] = useState(true);\n  const [payment, setPayment] = useState(null);\n  const [error, setError] = useState(null);\n\n  useEffect(() => {\n    const reference = searchParams.get('reference');\n    if (reference) {\n      verifyPayment(reference);\n    }\n  }, [searchParams]);\n\n  const verifyPayment = async (reference) => {\n    try {\n      const response = await fetch(\n        `http://localhost:5000/api/payments/verify/${reference}`\n      );\n      const result = await response.json();\n\n      if (result.success) {\n        setPayment(result.data);\n      } else {\n        setError(result.message);\n      }\n    } catch (err) {\n      setError('Failed to verify payment');\n      console.error(err);\n    } finally {\n      setLoading(false);\n    }\n  };\n\n  if (loading) return <div>Verifying payment...</div>;\n\n  if (error) {\n    return (\n      <div className=\"payment-failed\">\n        <h2>Payment Verification Failed</h2>\n        <p>{error}</p>\n      </div>\n    );\n  }\n\n  if (payment && payment.status === 'completed') {\n    return (\n      <div className=\"payment-success\">\n        <h2>✅ Payment Successful!</h2>\n        <div className=\"details\">\n          <p>Amount: ₦{payment.amount.toLocaleString()}</p>\n          <p>Reference: {payment.reference}</p>\n          <p>Tenant: {payment.tenant.name}</p>\n        </div>\n      </div>\n    );\n  }\n\n  return <div>Payment verification incomplete</div>;\n}\n```\n\n---\n\n## Configuration\n\n### Development Setup\n\n1. **Update `.env` file**:\n   ```env\n Cancel: http://localhost:8080/payment-cancelled\n   ```\n\n3. **CORS Configuration** (already set in `.env.example`):\n   ```env\n   ALLOWED_ORIGINS=http://localhost:8080\n   ```\n\n### Production Common Scenarios\n\n```javascript\n// Scenario 1: Reference not provided\nGET /api/payments/verify/\n→ 400 Bad Request: \"Payment reference is required\"\n\n// Scenario 2: Payment record not found\nGET /api/payments/verify/INVALID-REF\n→ 404 Not Found: \"Payment record not found\"\n\n// Scenario 3: Paystack verification fails\nGET /api/payments/verify/PAY-xxx\n→ 400 Bad Request: \"Failed to verify payment with Paystack\"\n\n// Scenario 4: Network timeout\n→ Frontend should retry with exponential backoff\n\n// Scenario 5: Payment abandoned\n→ Payment remains \"pending\" in database\n→ Can be retried by initiating new payment\n```\n\n---\n\n## Best Practices\n\n### Frontend\n\n✅ **Do:**\n- Store payment reference in localStorage for recovery\n- Implement automatic retry with exponential backoff\n- Show loading state while verifying\n- Display clear success/failure messages\n- Redirect to dashboard after verification\n- Handle network timeouts gracefully\n- Log verification attempts for debugging\n\n❌ **Don't:**\n- Assume payment is successful without verification\n- Perform duplicate verifications immediately\n- Log sensitive payment details to console\n- Redirect before verification completes\n- Cache verification results\n\n### Backend\n\n✅ **Do:**\n- Always verify with Paystack API\n- Update payment status immediately\n- Log all verification attempts\n- Return complete payment details\n- Handle Paystack API timeouts\n- Return appropriate HTTP status codes\n\n❌ **Don't:**\n- Trust frontend-provided payment status\n- Skip Paystack verification\n- Update database before verification\n\n---\n\n## Testing Verification\n\n### Development Flow\n\n1. **Start backend**:\n   ```bash\n   npm run dev\n   ```\n\n2. **Initiate payment**:\n
