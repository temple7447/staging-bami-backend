# Payment Verification & Dashboard Callback Flow\n\n## Overview\n\nThe payment system now supports **automatic redirect to dashboard** after Paystack checkout, along with traditional API verification. When a payment is completed on Paystack, users are automatically redirected to `/dashboard/payment/success` with the payment reference and status.\n\n## Flow Diagram\n\n```\n┌─────────────────────────────────────────────────────────────────┐\n│                      PAYMENT INITIATION                          │\n│  POST /api/payments/{type} → Creates payment + Paystack URL    │\n└────────────────────┬────────────────────────────────────────────┘\n                     │\n                     ▼\n        ┌────────────────────────────┐\n        │  Frontend Receives:         │\n        │  - paymentLink (Paystack)   │\n        │  - reference (tracking)     │\n        │  - accessCode               │\n        └────────┬───────────────────┘\n                 │\n                 ▼\n        ┌────────────────────────────┐\n        │  Tenant Clicks Payment Link │\n        │  (Opens Paystack Checkout) │\n        └────────┬───────────────────┘\n                 │\n                 ▼\n        ┌────────────────────────────┐\n        │  Paystack Checkout Modal   │\n        │  (Tenant enters card info) │\n        └────────┬───────────────────┘\n                 │\n        ┌────────┴────────┐\n        │                 │\n        ▼                 ▼\n    SUCCESSFUL         FAILED\n    (Payment OK)   (Payment Declined)\n        │                 │\n        └────────┬────────┘\n                 │\n                 ▼\n        ┌─────────────────────────────────────────┐\n        │  Paystack Redirects to Callback URL:    │\n        │  /api/payments/verify/{reference}      │\n        │  ?redirect=true                         │\n        │                                          │\n        │  (Automatically included by Paystack)   │\n        └────────┬────────────────────────────────┘\n                 │\n                 ▼\n        ┌─────────────────────────────────────────┐\n        │  Backend Verifies Payment:              │\n        │  1. Check Paystack status               │\n        │  2. Update payment record               │\n        │  3. Trigger 50/30/20 distribution       │\n        │  4. Log transaction                     │\n        └────────┬────────────────────────────────┘\n                 │\n        ┌────────┴────────┐\n        │                 │\n        ▼                 ▼\n    SUCCESS           ERROR/FAILED\n        │                 │\n        └────────┬────────┘\n                 │\n                 ▼\n        ┌─────────────────────────────────────────┐\n        │  Backend Redirects Browser to:          │\n        │  /dashboard/payment/success?            │\n        │  reference={ref}                        │\n        │  &trxref={ref}                          │\n        │  &status={success|failed|error}         │\n        │  &message={optional}                    │\n        └────────┬────────────────────────────────┘\n                 │\n                 ▼\n        ┌─────────────────────────────────────────┐\n        │  Frontend Dashboard Page Loads:         │\n        │  - Displays payment result              │\n        │  - Shows transaction details            │\n        │  - Wallet balance updates               │\n        │  - Distribution breakdown               │\n        └─────────────────────────────────────────┘\n```\n\n## API Endpoints\n\n### 1. Initiate Payment (Any Type)\n\n**Request:**\n```http\nPOST /api/payments/{type}\nContent-Type: application/json\nAuthorization: Bearer {token}\n\n{\n  \"tenantId\": \"tenant_id\",\n  \"amount\": 100000,\n  \"description\": \"Optional description\"\n}\n```\n\n**Response:**\n```json\n{\n  \"success\": true,\n  \"message\": \"Rent payment initiated successfully\",\n  \"data\": {\n    \"paymentId\": \"payment_id\",\n    \"paymentLink\": \"https://checkout.paystack.com/...\",\n    \"reference\": \"PAY-1234567890-abc123def456\",\n    \"accessCode\": \"access_code_123\",\n    \"amount\": 100000,\n    \"type\": \"rent\",\n    \"tenant\": {\n      \"name\": \"John Doe\",\n      \"unit\": \"Unit A\"\n    }\n  }\n}\n```\n\n**Frontend Action:**\n```javascript\n// After receiving response, redirect user to Paystack\nwindow.location.href = paymentResponse.data.paymentLink;\n\n// Or use Paystack inline form\nPaystackPop.setup({\n  key: PAYSTACK_PUBLIC_KEY,\n  email: tenant.email,\n  amount: paymentResponse.data.amount * 100, // Kobo\n  ref: paymentResponse.data.reference,\n  onClose: () => console.log('Payment cancelled'),\n  onSuccess: (response) => console.log('Payment successful')\n}).openIframe();\n```\n\n### 2. Verify Payment (Browser Redirect)\n\n**Automatic Redirect from Paystack:**\n```\nGET /api/payments/verify/{reference}?redirect=true\n```\n\n**Backend Processing:**\n1. Finds payment record by reference\n2. Queries Paystack for verification\n3. Updates payment status (completed/failed)\n4. **Triggers 50/30/20 distribution** if successful\n5. Redirects browser to dashboard\n\n**Redirect URL Generated:**\n\n**On Success:**\n```\nGET /dashboard/payment/success?\n  reference=PAY-1234567890-abc123def456\n  &trxref=PAY-1234567890-abc123def456\n  &status=success\n```\n\n**On Failure:**\n```\nGET /dashboard/payment/success?\n  reference=PAY-1234567890-abc123def456\n  &status=failed\n  &message=Payment+failed\n```\n\n**On Error:**\n```\nGET /dashboard/payment/success?\n  reference=PAY-1234567890-abc123def456\n  &status=error\n  &message=Error+verifying+payment\n```\n\n### 3. Verify Payment (API Call)\n\n**Request:**\n```http\nGET /api/payments/verify/{reference}\nAuthorization: Bearer {token}\n```\n\n**Response:**\n```json\n{\n  \"success\": true,\n  \"message\": \"Payment verification successful - Status: success\",\n  \"data\": {\n    \"paymentId\": \"payment_id\",\n    \"reference\": \"PAY-1234567890-abc123def456\",\n    \"status\": \"completed\",\n    \"paystackStatus\": \"success\",\n    \"amount\": 100000,\n    \"currency\": \"NGN\",\n    \"type\": \"rent\",\n    \"paidAt\": \"2024-01-01T10:30:00Z\",\n    \"tenant\": {\n      \"name\": \"John Doe\",\n      \"email\": \"john@example.com\",\n      \"unit\": \"Unit A\"\n    },\n    \"estate\": {\n      \"name\": \"Sunset Estate\"\n    }\n  }\n}\n```\n\n## Paystack Configuration\n\n### Callback URL Setup\n\nThe callback URL is **automatically generated** by the backend:\n\n```\n{BACKEND_URL}/api/payments/verify/{reference}?redirect=true\n```\n\n**Environment Variables Required:**\n```env\nBACKEND_URL=https://api.bamihost.com          # Production\n# or\nBACKEND_URL=http://localhost:5000             # Development\n```\n\n### In Paystack Dashboard\n\n1. Go to **Settings** → **API Keys & Webhooks**\n2. Under **Webhooks**, set **POST URL**:\n   ```\n   {BACKEND_URL}/api/payments/callback\n   ```\n3. The callback redirect is handled per-transaction via metadata\n\n## Frontend Implementation\n\n### Success Page Component\n\n```jsx\n// /dashboard/payment/success.jsx\n\nfunction PaymentSuccessPage() {\n  const searchParams = new URLSearchParams(window.location.search);\n  \n  const reference = searchParams.get('reference');\n  const trxref = searchParams.get('trxref');\n  const status = searchParams.get('status');\n  const message = searchParams.get('message');\n  \n  useEffect(() => {\n    if (status === 'success') {\n      // Call API to fetch payment details\n      fetchPaymentDetails(reference || trxref);\n      // Refresh wallet balance\n      refreshWalletBalance();\n      // Show success toast\n      toast.success('Payment completed successfully!');\n    } else if (status === 'failed') {\n      toast.error('Payment failed. Please try again.');\n    } else if (status === 'error') {\n      toast.error(message || 'An error occurred during payment verification.');\n    }\n  }, [status]);\n  \n  return (\n    <div className=\"payment-success-page\">\n      {status === 'success' && (\n        <div className=\"success-content\">\n          <h1>✅ Payment Successful</h1>\n          <p>Reference: {reference}</p>\n          <p>Your payment has been verified and the 50/30/20 distribution has been applied.</p>\n          <button onClick={() => window.location.href = '/dashboard'}>Back to Dashboard</button>\n        </div>\n      )}\n      \n      {(status === 'failed' || status === 'error') && (\n        <div className=\"error-content\">\n          <h1>❌ Payment Failed</h1>\n          <p>{message || 'There was an issue processing your payment.'}</p>\n          <button onClick={() => window.location.href = '/dashboard/payments'}>Try Again</button>\n        </div>\n      )}\n    </div>\n  );\n}\n\nexport default PaymentSuccessPage;\n```\n\n## Backend Implementation Details\n\n### Payment Controller - verifyPayment Function\n\n**Key Features:**\n\n1. **Dual Mode Support:**\n   - Browser redirect mode (when `?redirect=true` is present)\n   - API mode (when called without redirect parameter)\n\n2. **Automatic Distribution:**\n   - 50% → Marketing account\n   - 30% → Owner account\n   - 20% → Operations account\n\n3. **Proper Error Handling:**\n   - Payment not found → Redirect with error status\n   - Paystack verification failed → Redirect with error status\n   - Distribution error → Logged but payment still completes\n\n4. **Status Parameters:**\n   - `success` - Payment completed and distributed\n   - `failed` - Payment failed at Paystack\n   - `error` - System error during verification\n\n### Paystack Service - Callback URL Generation\n\n```javascript\n// Auto-generated callback URL includes redirect parameter\nconst callbackUrl = paymentData.callbackUrl || \n  `${process.env.BACKEND_URL}/api/payments/verify/${reference}?redirect=true`;\n```\n\n## Query Parameters Reference\n\n| Parameter | Type | Required | Description |\n|-----------|------|----------|-------------|\n| `reference` | string | Yes | Payment reference generated by backend |\n| `trxref` | string | No | Alternative reference (Paystack compatibility) |\n| `status` | string | Yes | `success`, `failed`, `error` |\n| `message` | string | No | Error message (URL encoded) |\n| `redirect` | string | No | Internal: tells API to redirect browser |\n\n## Testing the Flow\n\n### Test 1: Successful Payment\n\n```bash\n# 1. Record Not Found\n\n```\nPaystack calls: /api/payments/verify/invalid-reference?redirect=true\n↓\nBackend: Payment not found in database\n↓\nRedirects to: /dashboard/payment/success?reference=invalid-reference&status=error&message=Payment%20record%20not%20found\n```\n\n### Scenario 2: Paystack Verification Failed\n\n```\nPaystack calls: /api/payments/verify/{reference}?redirect=true\n↓\nBackend: Paystack API returned error\n↓\nRedirects to: /dashboard/payment/success?reference={ref}&status=error&message=Failed%20to%20verify%20payment\n```\n\n### Scenario 3: Payment Status Not Success\n\n```\nPaystack calls: /api/payments/verify/{reference}?redirect=true\n↓\nBackend: Paystack status is \"abandoned\" or \"cancelled\"\n↓\nRedirects to: /dashboard/payment/success?reference={ref}&status=failed&message=Payment%20abandoned\n```\n\n## Environment Setup\n\n**Required .env variables:**\n```env\n# Payment
